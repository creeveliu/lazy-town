# 在线电影日历设计

## 目标

在现有“游戏 / 电影”双 tab 基础上增加第三个 tab：`在线电影`。

`电影` tab 改名为 `院线电影`，继续展示豆瓣即将上映数据。`在线电影` 只收电影，不收剧集、综艺、纪录片；范围包括院线电影转线上、平台首发电影、网络电影、爱奇艺 / 腾讯视频 / 优酷等多平台同步上线影片。

## 数据原则

- 日历只输出有明确上线日期的影片。
- “即将上线”“敬请期待”等无日期内容可以暂不入库，或入库但不生成 ICS。
- 同一影片在多个平台同日上线时合并平台字段。
- 每条数据保留来源链接和来源名称，方便人工追溯。
- 抓取失败不影响现有游戏和院线电影同步。

## 数据来源

优先级从高到低：

1. 平台官方频道页：优酷电影频道、爱奇艺电影/VIP/云影院相关页面、腾讯视频电影频道。
2. 平台官方公告或新闻稿：适合补充明确上线日期。
3. 可检索的第三方报道：只作为补充来源，置信度低于官方来源。

第一版优先实现稳定可解析来源，宁可少收，不做猜测日期。

## 数据模型

新增表：`online_movies`

- `source_url`：唯一来源链接
- `title`
- `online_date`
- `platforms`：TEXT[]
- `source_name`
- `cover_url`
- `status`
- `confidence`
- `updated_at`

建议索引：`online_date`。

## 后端流程

新增 `src/lib/online-movie-source.ts`：

- 抓取多个来源。
- 解析标题、上线日期、平台、封面、来源链接。
- 过滤无明确日期的数据。
- 按标题 + 日期合并多平台记录。
- 按 `online_date ASC` 排序。

更新 `src/lib/db.ts`：

- `ensureSchema()` 创建 `online_movies`。
- 新增 `getOnlineMoviesFromDb()`。
- `/api/sync` 同步游戏、院线电影、在线电影。
- 在线电影同步异常单独兜底，不阻断另外两类。

新增 `/api/calendar/online-movies`：

- 输出在线电影 ICS。
- 事件标题格式：`片名 上线 平台名`。
- 描述包含平台、来源名称、来源链接。

## 前端

更新 `TabbedContent`：

- tab：`游戏 / 院线电影 / 在线电影`
- 在线电影表格列：`海报 / 上线日 / 片名 / 平台 / 状态`
- 订阅弹窗增加 `在线电影上线表`

移动端沿用现有表格策略，优先保证文字可读，不增加排序器。
